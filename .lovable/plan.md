

## Полная переработка логики и структуры дашбордов

### Проблемы сейчас
1. **AI-блок непонятен** — "ИИ-Директор" с абстрактными рекомендациями, нет ясности что он делает
2. **Health Score** — метрика без объяснения, непонятно что измеряет
3. **ROMI** — пользователь говорит что его не будет
4. **Данные** — нужны визиты и продажи (кол-во), а не суммы
5. **Нет таск-менеджмента** — нужна возможность добавлять и фиксировать задачи для команды
6. **Кнопки-пустышки** — Quick Actions, "Принять рекомендации", "Обновить статусы" ни к чему не ведут

### Что сделаем

#### 1. Панель управляющего (`DashboardPM.tsx`) — полная перестройка

**KPI карточки** (заменяем):
- Проекты: кол-во активных
- Визиты за месяц: суммарно по всем проектам
- Продажи за месяц: кол-во (не сумма)
- Задачи: выполнено/всего

**Таблица проектов** — убираем ROMI, убираем MRR. Колонки:
- Проект | Статус (🟢🟡🔴) | Визиты | Продажи | Задачи | PM

**Статус проекта** = простая логика: зелёный если задачи выполняются и есть визиты, жёлтый если задачи отстают, красный если нет визитов или всё стоит.

**Убираем AI-Директор** полностью. Вместо него — блок **"Задачи команды"** с возможностью добавления.

**Блок задач команды:**
- Список задач с: текст, ответственный, дедлайн, статус (новая/в работе/готово)
- Форма добавления задачи: инпут текста, выбор исполнителя, кнопка "Добавить"
- Чекбокс для отметки выполнения
- `useState` для управления массивом задач (локальный стейт)

#### 2. Панель таргетолога (`DashboardTarget.tsx`) — чистка

**Quick Actions** — привязываем к навигации:
- "Создать кампанию" → пока toast "В разработке"
- "Контент-Завод" → `navigate('/content')`
- "CRM" → `navigate('/crm')`  
- "Кабинеты" → `navigate('/accounts')`

**Таблица кампаний** — оставляем как есть, рабочая.

**Алерты** — оставляем, но кнопку "Обновить статусы" делаем toast.

#### 3. Панель продаж (`DashboardSales.tsx`) — чистка

**KPI:** убираем "Ср. чек" (суммы нет). Заменяем на "Визиты" (кол-во за месяц).
Убираем "AI обработал" — непонятно. Заменяем на "Новых лидов" (кол-во).

**Воронка** — убираем суммы (value). Оставляем только кол-во на каждом этапе.

**Таблица лидов** — убираем AI Score (непонятно). Оставляем: Имя, Проект, Источник, Время, Статус.

#### 4. Sheets (детальные карточки) — чистка

**ProjectDetailSheet** — убираем ROMI, MRR. Добавляем визиты/продажи.
**LeadDetailSheet** — убираем AI Score блок. Оставляем контакт + действия.
**CampaignDetailSheet** — оставляем, рабочий.

#### 5. Главный хаб (`Dashboard.tsx`)
- Упрощаем описания ролей до 1 строки понятным языком

### Технические детали

- Задачи команды — `useState<Task[]>` с интерфейсом `{ id, text, assignee, status, createdAt }`
- Добавление через `Input` + `Select` (исполнитель) + `Button`
- Отметка выполнения через `Checkbox`
- Все кнопки Quick Actions либо навигируют, либо показывают toast
- Данные по визитам/продажам — моковые числа в таблицах
