

# CRM: от мок-данных к реальному управлению

## Текущее состояние

Все 4 вкладки CRM работают на захардкоженных данных:
- **Чаты**: `MOCK_CHATS` — статичные сообщения, не привязаны к лидам
- **Воронка (Канбан)**: единственная вкладка, работающая с реальной таблицей `leads`
- **База клиентов**: `MOCK_CLIENTS` — не связана с `leads`
- **Автоматизации**: `MOCK_AUTOMATIONS` — нет хранения в БД
- **LeadDetailSheet**: моковые сообщения, активность, заметки

## Что будет сделано

### 1. Новые таблицы в Supabase

**`crm_messages`** — хранение переписки с клиентами
```text
id, lead_id (FK leads), direction (inbound/outbound), 
sender_type (client/ai/manager), body, channel (whatsapp/instagram/web),
read boolean, created_at
```

**`crm_notes`** — заметки менеджеров по лидам
```text
id, lead_id (FK leads), author_name, body, created_at
```

**`crm_automations`** — сценарии автоматизации
```text
id, project_id, trigger_type, trigger_value, 
action_type, action_detail, is_enabled, created_at
```

RLS: все таблицы доступны авторизованным пользователям. Realtime включён для `crm_messages` и `crm_notes`.

### 2. Чаты — реальные данные

- Загрузка сообщений из `crm_messages` по `lead_id`
- Realtime-подписка на новые сообщения — мгновенное появление в списке
- Отправка сообщений менеджером записывается в `crm_messages`
- Счётчик непрочитанных считается из `crm_messages.read = false`
- Последнее сообщение в списке чатов — реальное из БД

### 3. База клиентов — из таблицы `leads`

- Убрать `MOCK_CLIENTS`, загружать из `leads` с агрегацией
- LTV = сумма `amount` лидов со статусом "Оплачен" по одному телефону
- Источник, AI-рейтинг, последний визит — из реальных данных
- CSV-экспорт генерирует файл из текущего фильтрованного списка

### 4. Автоматизации — CRUD через Supabase

- Загрузка/создание/редактирование/удаление из `crm_automations`
- Форма создания сценария: выбор триггера (смена этапа, таймер без ответа), действия (WhatsApp, уведомление)
- Переключатель вкл/выкл сохраняется в БД

### 5. LeadDetailSheet — реальные данные

- Чат внутри sheet загружает `crm_messages` по `lead_id`
- Заметки загружаются и сохраняются в `crm_notes`
- Лента активности формируется из `crm_messages` + изменений статуса `leads.updated_at`

### 6. Добавление нового лида

- Кнопка "+ Новый лид" в шапке CRM
- Форма: имя, телефон, источник, сумма, этап, проект
- Запись через `supabase.from("leads").insert()`

## Порядок реализации

1. Миграция: создать 3 таблицы + RLS + realtime
2. Переписать `ClientDatabase` на данные из `leads`
3. Переписать `ChatsView` — реальные сообщения из `crm_messages`
4. Переписать `LeadDetailSheet` — сообщения и заметки из БД
5. Переписать `Automations` — CRUD из `crm_automations`
6. Добавить форму создания нового лида
7. Добавить кнопку CSV-экспорта с реальными данными
